作者： 何茜悦

一、基本概念

  1. MVP(Minimum Viable Product，最小可用产品）来源于Eric Ries 的《精益创业》。
  2. 创业者或者新产品的创意用最简洁的方式开发出来。四个特点：创意得以展示、可以用来调整或者展示、简洁、开发低成本甚至零成本。
  3. 在低成本的前提下，将核心价值体现出来的最好方法。
  4. 存在的目的：快速验证产品是否满足了用户需求，是否能创造商业价值。

二、使用原因

  1. 好的产品不是“面面俱到”，而是实现核心价值。
  2. 产品模型的合理也不能确保功能也会受到用户认可，快速投入市场中进行验证是最妥当的方法。(低成本试错)
  3. 产品的核心功能就能解决用户的问题(越快投入市场越快获得用户)
  4. 很多产品都是“从小做到大“。例如，微信

三、PMF（产品市场匹配点）

  1. 存在的目的：从市场的角度去定义一个产品。
  2. 在一个好的市场里，能够用一个产品去满足这个市场。
  3. PMF理论：产品的增长曲线会在找到契合的这个点之后，快速增长。在这之前，一直是在较低范围的波动状态。

四、设计MVP

1、方法

  1. 奥卡姆剃刀法；
  2. 去掉可人工处理的部分；
  3. 用户访谈；
  4. 确保只有一个功能。

2、要求

  1. 达到可用与最小成本的平衡；
  2. 不能臃肿：会导致成本过高；
  3. 不能简陋：到达残缺的地步，会影响用户的正常使用方式；

3、具体实现

  1. 广告：测验有多少陌生人对这个产品产生兴趣，再收集用户的反馈；

    Dropbox：https://v.qq.com/x/page/x0134u6ay87.html

  2. 假MVP：视觉效果没问题，但部分功能是假的；

    Sendwithus：http://zero.sendwithus.com/gallery/

  3. 线下实现:在线下摸清模式运作后，再考虑移植到线上;

  4. 众筹：产品的功能设想预售，同时检验功能和商业价值的方法。

    一般使用在硬件，原因是：绝大部分的硬件众筹，并不是从0开始，而是已经开模、设计、调研、生产，至少已经落地。硬件众筹现在更多的含义是：以一个低于零售价的价位，来进行提前的市场内测观察，更像是一个抢先版/内测版，众筹只是量产。而软件大部分还只是从0开始。

五、基于MVP思想的敏捷开发

1、敏捷开发

  1. 事实上，很多产品在开发中总是喜欢大爆炸交付：直到100%完成了才构建产品并在项目结束时交付；
  2. 而这幅图是一种替代方式：敏捷开发，可是还是会出现上面这种错误的迭代方式。
  3. 背景：我们的客户想要一台汽车；
  4. 但是我们要发掘潜在需求：客户可能是想快速地从A到B，也许还有其他的需求，但是我们得一步一步的去学习。

2、第一行

  1. 产品在迭代过程中始终处于不可用状态；
  2. 没有用户体验的反馈，产品中很可能充斥着人们基于对用户需求的错误假设而产生的设计缺陷；
  3. 在现实中这通常不成立。从技术上看，它可能是增量和迭代交付的，但使得产品开发变得极具风险。

3、第二行

  1. 第一个发布版本称为“滑板版本”。

    客户不满意？这就是重点——我们并不试图让客户在此时就感到满意。

    我们此时的主要目标只是学习。

  2. 迭代版本：做到“不会摔”

    • 思考：客户现在满意了吗？
    • 或许还没有，他还是想要一台汽车，但是我们确实收到了用户的真实反馈。

  3. 也许会迭代：“自行车”版本，自行车还不够快，那就加上引擎，可以去到更远的地方，又快又轻松。

  4. 也许会停止迭代。

    不过更可能的是，我们基于用户的反馈，迭代出更好的车。

4、从反面思考：

  1. 大爆炸交付：真的就不能考虑吗？是十年磨一剑？还是错过时代的进步。
  2. 敏捷开发：从开发者角度来说，确实能够带来大量学习的机会，并且能够正确的抓住用户的痛点，甚至是痒点。从用户角度考虑，大量的半成品产品充斥市场，会造成什么现象

5、适用原则

  1. 适用于软件，硬件成本高，达不到快速迭代；
  2. 用户忍受程度很低时，不建议敏捷开发；
  3. 用户量很大，或者是市场还未开拓时，可以大爆炸交付，以抢占先机；
  4. 敏捷开发适用于快速变化、竞争激烈的市场；
  5. 你的客户也不知道自己需要什么时，使用敏捷开发。

六、微信迭代史

  1. 2011年1月:1.0版本：一个在熟人之间发免费文本和图片的工具；

  2. 2011年5月：2.0版本：语音功能和“查找附近陌生人”的功能；

  3. 2.5版本：上线功能：“附近的人”，正式成为陌生人社交；已经甩开竞争对手。

  4. 2011年10月：3.0版本：“扫一扫”、“服务号”；

  5. 3.5版本、3.6版本：上线功能：“扫描二维码”、“微信的公众号”，为朋友圈的内容做准备，开通API接口；

  6. 2012年5月：4.0版本：微信推出了“朋友圈”；

  7. 4.2版本朋友圈的评论功能上线，说明功能并不是需要一下子交付；

  8. 4.5版本：优化功能：“内容公众号”，正式引入自媒体；

  9. 2013年8月：5.0版本：服务号与支付号、微信支付。

    用户增长：“打飞机”游戏，微信红包的发布，与春晚的合作。
